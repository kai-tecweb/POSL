# 🟦 POSL V1.0 UI設計書（完全まとめ版）

POSL（ポスエル）は「あなたの分身が書いたような、自然で前向きなX投稿」を毎日20時に自動生成・投稿するシステム。
その中核となるのがこの UI設計。

各画面が互いに役割を補い、
「投稿の性格」「話し方」「文章の型」「人格」「イベント性」「トレンド感」
をバランスよく調整できるようになっている。

---

## 1. Dashboard（ダッシュボード）
- 画面の目的：毎日の状況がひと目で分かる"司令塔"
- 要素一覧：今日の投稿予定、今週の曜日テーマ、最新の日記（人格データ）、今日のトレンドTOP3、次のイベント、API接続状況、自動投稿状況
- 主な遷移：自動投稿設定、イベント管理、曜日テーマ、トレンド設定、人格データ、投稿ログ

## 2. 自動投稿設定
- 目的：投稿の時間と頻度を決める、POSLの“心臓部”
- UI内容：自動投稿 ON/OFF、投稿時間（時刻ピッカー）、1日1投稿固定（V1方針）、テスト投稿ボタン、API接続の状態（X / OpenAI）

## 3. 曜日テーマ設定
- 目的：毎日の文章の“方向性”を定める場所
- UI内容：曜日ごとのテーマ名・説明編集、プレビュー文章生成

## 4. イベント管理
- 目的：「特別な日」に一本追加するか、差し替えるかを決める
- UI内容：イベント一覧（ON/OFF）、種別（固定/マイイベント）、日付、挙動（追加/置き換え）、テンプレ選択、文体の上書き、補足メモ、プリセット追加

## 5. トレンド設定
- 目的：投稿に“今の空気感”を混ぜる割合を調整
- UI内容：トレンド利用ON/OFF、取得元（Google/Yahoo等）、混ぜ方スライダー、混ぜ方スタイル、避けたいカテゴリ、プレビュー

## 6. 文体・トーン設定
- 目的：POSLの“話し方・口調・人柄”を決める
- UI内容：文体プロファイル選択、スライダー（丁寧さ/カジュアルさ/ポジティブ度/専門っぽさ/感情の濃さ/比喩の量/絵文字使用度）、禁止・注意表現、よく使いたい表現、プレビュー、デフォルトリセット

## 7. 投稿テンプレ（型）設定
- 目的：文章の“骨格”を決める（レシピ）
- UI内容：テンプレ一覧、有効/無効、優先度、説明、詳細表示、代表テンプレ例、構成リセット

## 8. プロンプト微調整
- 目的：AIへの"追加ルール・こだわり"を伝える（文体設定とは別の補強設定です）
- UI内容：ベースプロンプト（読み取り専用）、追加ルール入力、NGワード、よく使ってほしい表現、AI自由度スライダー、デフォルトに戻す

## 9. 音声日記アップロード
- 目的：人格データベースを育てる"日記口"
- UI内容：録音ボタン、ファイルアップロード、変換ステータス、日記一覧（再生ボタン付き）、感情タグ、人格反映ON/OFF、日記詳細表示、削除

## 10. 人格データ閲覧
- 目的：あなたの日記や経験から抽出した"人格プロフィール"を確認し、POSLがどのようにあなたらしさを学習したかがわかる画面（この画面は編集不可の読み取り専用です）
- UI内容：人格要約、性格傾向（棒グラフ）、よく使う表現、最近の感情トレンド、AIの解釈

## 11. 投稿ログ
- 目的：実際に投稿されたXの履歴を一覧
- UI内容：投稿日時、用いたテンプレ、文体情報、生成テキスト全文、使用したプロンプト（内部検証用）、投稿ステータス、Xリンク

## 12. API設定（X / OpenAI）
- 目的：システムが正しく動くための“鍵”設定
- UI内容：X API（API Key/Secret/Access Token/状態チェック）、OpenAI API Key、接続テストボタン

## 13. 全画面の関係（遷移図 要約）
Dashboard
↳ 曜日テーマ
↳ イベント管理
↳ トレンド設定
↳ 人格データ
↳ 投稿ログ
↳ 設定（API / 文体 / テンプレ / プロンプト微調整）

POSLのUIは「ホーム → 詳細設定 → 戻る」が迷わず辿れる構造。

---

## まとめ（ポイント）
- POSLのUIは“専門的だけどやさしい”をテーマに設計
- 設定項目は自然言語ベースで、迷わせない構造
- すべての画面が「日々の投稿」をより自然に、人格的に、前向きにするためのもの
- UIは開発者がすぐ動けるレベルに整理済み
